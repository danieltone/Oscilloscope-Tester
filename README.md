# Oscilloscope Tester — Raspberry Pi Pico

A MicroPython firmware that turns a Raspberry Pi Pico into a multi-waveform
signal generator for bench-testing and calibrating an oscilloscope.

---

## Features

| Category | Test signals |
|----------|-------------|
| **Square waves** | 1 Hz, 10 Hz, 100 Hz, 1 kHz, 10 kHz, 100 kHz, 1 MHz |
| **PWM duty cycle** | 10 %, 25 %, 50 %, 75 %, 90 % at 1 kHz |
| **Sine wave** | 1 Hz, 10 Hz, 100 Hz (DDS via PWM) |
| **Triangle wave** | 10 Hz, 100 Hz (DDS via PWM) |
| **Sawtooth wave** | 10 Hz, 100 Hz — rising and falling (DDS via PWM) |
| **DC levels** | 0 V, ~0.83 V, ~1.65 V, ~2.48 V, 3.3 V |
| **Burst mode** | 1 kHz and 10 kHz with 10 cycles ON / 10 cycles OFF |
| **Frequency sweep** | 1 Hz → 1 MHz, log-spaced, 2 s per step |

All signals appear on **GP0 (physical pin 1)**. GND connects to any GND pin.

---

## Hardware Requirements

- Raspberry Pi Pico (or Pico W) with MicroPython installed
- Oscilloscope + probe
- *(Optional for analog waveforms)* — RC low-pass filter components:
  - 1 × 10 kΩ resistor
  - 1 × 10 nF capacitor (ceramic)
- Breadboard and jumper wires

## RC Low-Pass Filter (for analog waveforms)

Analog waveforms (sine, triangle, sawtooth, DC levels) are generated by
rapidly varying the duty cycle of a 62.5 kHz PWM carrier. Without a filter
the oscilloscope sees the 62.5 kHz carrier with a changing duty cycle —
visible but not a clean waveform shape.

Adding a simple RC filter between the Pico output and the scope probe tip
removes the carrier and reveals the underlying waveform:

```
GP0 (pin 1)
    |
   [10 kΩ]
    |
    +---------> to scope probe tip
    |
   [10 nF]
    |
   GND
```

**Cut-off frequency:** f_c = 1 / (2π × 10 kΩ × 10 nF) ≈ **1.6 kHz**

This filters out the 62.5 kHz carrier while passing waveforms up to ~100 Hz
with less than 0.4 dB of attenuation.

> **Tip:** Many oscilloscopes have a built-in 20 MHz bandwidth-limit mode
> (BW Limit button). Enabling it will reduce noise but won't remove a
> 62.5 kHz carrier. The external RC filter is the cleanest solution.

## Software Setup

### 1. Install MicroPython on the Pico (skip if already done)

1. Download the latest MicroPython UF2 from
   <https://micropython.org/download/RPI_PICO/>
2. Hold the **BOOTSEL** button on the Pico and plug it into USB.
3. It mounts as a USB drive — copy the `.uf2` file onto it.
4. The Pico reboots into MicroPython automatically.

### 2. Install mpremote (upload tool)

```bash
pip install mpremote
```

### 3. Upload the firmware

```bash
git clone <this-repo>    # or copy files manually
cd Oscilloscope-Tester
chmod +x deploy.sh
./deploy.sh              # auto-detects /dev/ttyACM0
# or: ./deploy.sh /dev/ttyACM1
```

The script uploads `waveforms.py` and `main.py`, then opens the serial
terminal. Alternatively use **Thonny IDE** to copy the two files.

### 4. Manual upload (Thonny)

1. Open Thonny → select **MicroPython (Raspberry Pi Pico)** interpreter.
2. Open `waveforms.py` → *File → Save as → Raspberry Pi Pico* → `waveforms.py`
3. Repeat for `main.py`.
4. `main.py` runs automatically each time the Pico boots.

## Usage

Open a serial terminal at **115200 baud**:

```bash
mpremote connect /dev/ttyACM0      # mpremote (interactive REPL + terminal)
screen /dev/ttyACM0 115200         # screen
minicom -b 115200 -D /dev/ttyACM0  # minicom
```

The menu appears on the terminal:

```
+------------------------------------------------------+
|       OSCILLOSCOPE TESTER  --  Raspberry Pi Pico     |
|   Probe: GP0 (pin 1)   GND: any GND pin              |
+------------------------------------------------------+

 -- SQUARE WAVES  (hardware PWM, probe directly) -------
  [1]   1 Hz        [2]   10 Hz       [3]  100 Hz
  ...
Choice:
```

![Oscilloscope Tester menu](menu.png)

Type a number and press **Enter** to start the waveform. The signal runs
continuously until you press **any key** to stop it, then the menu returns.

## How It Works

### Square waves and PWM (choices 1–12)

The RP2040 hardware PWM peripheral generates the signal with hardware
accuracy. The CPU is free to monitor the serial port for a stop command.
Frequency range: 8 Hz – ~62.5 MHz.

### Analog waveforms — DDS (choices 13–20)

Direct Digital Synthesis (DDS) is used:

1. A lookup table with the waveform shape (sine, triangle, etc.) is
   pre-calculated at boot (256, 64, or 16 samples depending on frequency).
2. The PWM duty cycle is updated at `frequency × samples_per_cycle`
   samples/second using a busy-wait timing loop.
3. The 62.5 kHz PWM carrier is modulated by the duty cycle; an RC filter
   recovers the analog envelope.

| Requested frequency | Samples used | Sample rate |
|---------------------|-------------|-------------|
| ≤ 10 Hz             | 256         | ≤ 2560 Hz   |
| ≤ 100 Hz            | 64          | ≤ 6400 Hz   |
| ≤ 1000 Hz           | 16          | ≤ 16000 Hz  |

### Burst mode (choices 26–27)

Software GPIO toggling with busy-wait loops produces the ON burst; a timed
delay produces the OFF gap. Useful for checking oscilloscope trigger hold-off
and timebase accuracy.

### Frequency sweep (choice 28)

Steps through frequencies on a base-10 logarithmic scale (∼octave steps)
from 1 Hz to 1 MHz, holding each for 2 seconds. Useful for checking
oscilloscope bandwidth and frequency-response roll-off.

## Troubleshooting

| Symptom | Cause / Fix |
|---------|------------|
| No signal on scope | Check probe on GP0 (pin 1), clip on GND (pin 3) |
| Waveform looks like fast PWM square wave | Normal without RC filter for analog signals |
| Sine/triangle looks jagged | Increase scope time/div; or add RC filter |
| Menu does not appear | Check baud rate (115200). Try pressing Enter once. |
| `ImportError: no module named 'waveforms'` | `waveforms.py` missing — upload it |
| High-frequency outputs (100 kHz+) look rounded | Scope or probe bandwidth limit — expected |
